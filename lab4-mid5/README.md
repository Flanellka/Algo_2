# mid5 Определение максимальной RPS между микросервисами

### Описание

У вас есть платформа, состоящая из K микросервисов, представленных в виде графа

**Исток** - frontend, **сток** - общая бд между всеми (архитектор критин, сорян)

**Узел** - конкретный микросервер, связанный с другими
**Величина поток в ребре** - минимум между RPS двух соедененных микросервисов

Вам необходимо найти максимальный поток (RPS) в платформе любым из алгоритмов, и вычленить сервисы и пути, которые являются bottle-neck

### Ограничения
Ограничений нет, развлекайтесь. Тут вам надо пофантазировать, подумать над кейсами (реалистичными). От сюда и middle.


В отчете предоставить запуски для разных конфигураций (просто скрины и какой максимальный поток получился). Не парьтесь сильно над оформлением.

*Если в отчете будут однотипные графы (с заменой названий и потоком ребер), или без фантазии - то практик имеет право снизить баллы до **light.***
